roduct Requirements Document (PRD) dla FullStackQuiz
1. Wstęp i Cele
FullStackQuiz to aplikacja webowa/mobile do nauki programowania poprzez interaktywne quizy z pytaniami wielokrotnego wyboru. Cele: Umożliwić użytkownikom (deweloperom jak Ty z Polski) zdobywanie wiedzy z fullstack (Java/Spring, frontend), śledzenie postępów i rywalizację. Business: MVP w 3 miesiące, 1000 użytkowników w rok 1.

2. Użytkownicy i Persony
Primary User: Backend developer (np. Java/Spring expert z Warszawy), 25-40 lat, uczy się fullstack. Potrzeby: codzienne quizy, wyniki, leaderboards.
​

Admin: Ty jako twórca – dodaje/edytuje quizy.
Założenia: Użytkownicy mobilni (80%), polsko-angielski interfejs.

3. Funkcje Kluczowe
Lista Quizów: Przeglądanie quizów po tematach (Spring Security, React Hooks), wybór rozmiaru (5/10/20 pytań).

Strona Quizu: Jedno pytanie na ekran, 4 opcje, timer, progress bar, nawigacja prev/next.
​

Wyniki: Natychmiastowe po submit, procent poprawnych, słabe obszary, historia quizów.

Dashboard: Statystyki, wykresy (Chart.js), leaderboards.

Admin Panel: CRUD quizów/pytan (import JSON/Excel), zarządzanie użytkownikami.

Autentykacja: Login/register (JWT), role user/admin.

4. User Stories
Jako user, chcę wybrać quiz i odpowiedzieć na pytania wielokrotnego wyboru, by przetestować wiedzę.

Jako user, po quizie widzę wyniki z wyjaśnieniami błędów i sugestie nauki.

Jako admin, dodaję nowe quizy z pytaniami i opcjami, by rozszerzać bibliotekę.

Jako user, loguję się i widzę swój postęp w dashboardzie z wykresami.

5. Design i UI/UX
Dark mode, responsywny (Bootstrap/Tailwind), glassmorphism. Layout jak opisano wcześniej: header z progressem, duże pytanie, karty opcji. Prototyp w Figma: [link do szkicu].

6. Wymagania Techniczne
Backend: Spring Boot, JPA (Oracle/PostgreSQL), REST API, JWT auth. Stack: Java 21, Spring Security.
​
Frontend: React/Next.js 18, Chart.js dla wykresów.
Baza: Encje: User, Quiz, Question, Answer, UserQuizResult.
Inne: Docker/K8s deploy, PWA dla mobile.

Komponent	Technologia	Priorytet
API Quiz	Spring Boot REST	Wysoki
Strona Quizu	React Components	Wysoki
Dashboard	Chart.js	Średni
Admin Panel	React + Forms	Wysoki
7. Nie-Funkcjonalne Wymagania
Wydajność: <2s ładowanie strony, 1000 concurrent users.

Bezpieczeństwo: HTTPS, rate limiting, SQL injection protection.

Dostępność: WCAG 2.1, polski/EN.

Skalowalność: Cloud (AWS/Heroku).
​

8. Success Metrics i Timeline
Metrics: 70% retention po 30 dniach, NPS >8, 50% quizów ukończonych.

Timeline: MVP (auth + basic quiz) – 4 tyg.; Full release – 12 tyg.; Launch Q2 2026.
​

9. Ryzyka i Założenia
Ryzyka: Opóźnienia w AI-generowaniu pytań (opcjonalne). Założenia: Używasz Spring/K8s, start
