User Stories
Format: "Jako [rola], chcę [funkcja], aby [korzyść]". Z acceptance criteria.

Epicki User Stories (Główne):

Jako user, chcę zarejestrować konto i zalogować się, aby zapisywać moje wyniki i postęp.

AC: Email/password, JWT token, remember me.

Jako user, chcę wybrać quiz po temacie (np. Spring), aby uczyć się fullstack.

AC: Filtry po tech, podgląd pytań.

Jako user, chcę przejść quiz z timerem i zaznaczać odpowiedzi, aby symulować egzamin.

AC: Jedno pytanie/ekran, progress bar, feedback po odpowiedzi.

Szczegółowe User Stories:

Jako user, chcę zobaczyć natychmiastowy wynik po każdym pytaniu, aby zrozumieć błędy.

AC: Zielony/czerwony feedback, wyjaśnienie poprawnej odpowiedzi.
​

Jako user, chcę udostępnić wyniki w social media, aby rywalizować z przyjaciółmi.

AC: Share button (FB, LinkedIn), screenshot z wynikiem.

Jako admin, chcę importować pytania z JSON/CSV, aby szybko dodawać nowe quizy.

AC: Walidacja formatu, podgląd przed zapisem.

Jako user, chcę codzienne wyzwanie (daily quiz), aby utrzymywać rutynę nauki.

AC: Push notifications (PWA), streak counter.

Jako admin, chcę zobaczyć analityki (np. najtrudniejsze pytania), aby poprawiać quizy.

AC: Wykresy % poprawności per pytanie.

Te stories priorytetyzuj w backlogu (MVP: auth + basic quiz).
